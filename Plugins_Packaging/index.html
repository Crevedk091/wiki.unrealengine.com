# Plugins Packaging

Currently packaging process for plugins is a bit strange and deletes a lof of its fuctionality.

To prevent any classes will be removed (especially if you're focused blueprints), you can apply simple trick - you should "show" to the compilator that classes are used.

class FMyPlugin : public IMyPlugin { /** IModuleInterface implementation */ virtual void StartupModule() OVERRIDE { // @HACK Force classes to be compiled on shipping build UMyClassOne::StaticClass(); UMyClassTwo::StaticClass();   /** List all your classes here */ }   virtual void ShutdownModule() OVERRIDE {   } };

It's a trick, but it works.
